# 2026-03-13 実験記録
## TestFlight アップロードエラー修正 - アプリアイコン設定

### 概要 (Overview)
TestFlightアップロード時に3つのバリデーションエラーが発生していた問題を修正し、アップロード成功。
- Missing icon 120x120 (iPhone)
- Missing icon 152x152 (iPad)
- Missing CFBundleIconName in Info.plist

### 変更元ファイル (Source file)
- `Annotty-MRI/Info.plist`
- `Annotty-MRI/Assets.xcassets/AppIcon.appiconset/Contents.json`
- `Annotty-MRI/Assets.xcassets/AppIcon.appiconset/` (全18サイズのアイコンPNG新規追加)
- `Annotty-MRI.xcodeproj/project.pbxproj` (ビルド設定)

### Before / After 比較
| 項目 | Before | After |
|------|--------|-------|
| アイコンPNG | 存在しない | ChatGPTで生成した近未来風眼窩MRIアイコン、全18サイズ |
| Contents.json | universal 1024x1024のみ（filename未指定） | iPhone/iPad/Marketing全サイズを個別エントリで記載 |
| Info.plist | CFBundleIconName なし | `CFBundleIconName: AppIcon` を追加 |
| project.pbxproj | INFOPLIST_KEY_CFBundleIconName なし | Debug/Release両方に追加 |
| project.pbxproj | INCLUDE_ALL_APPICON_ASSETS なし | Debug/Release両方に `YES` で追加 |

### 変更理由 (Reason for change)
TestFlight/App Store Connectのバリデーションを通過するため。

### ハマった点と解決策 (Troubleshooting)
1. **Gemini CLIは画像生成非対応** → ChatGPTで近未来風MRIアイコンを生成
2. **単一サイズ(universal)のContents.jsonでは不十分だった** → 全18サイズの個別PNGを生成しContents.jsonに各エントリを記載して解決
3. **`GENERATE_INFOPLIST_FILE = YES` の場合、Info.plistへの直接記載だけでは反映されない** → `project.pbxproj`に `INFOPLIST_KEY_CFBundleIconName = AppIcon` と `ASSETCATALOG_COMPILER_INCLUDE_ALL_APPICON_ASSETS = YES` を追加して解決
4. **アルファチャンネルありPNGはリジェクトされる** → Pillowで検証しRGBであることを確認

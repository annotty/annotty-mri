# 2026-03-17 実験記録

## HITL マルチクラストレーニングパイプライン構築

### 概要 (Overview)
Server内のvanilla_unet_resnet34モデルをiPadと共有し、iPadからtrainingボタン押下でHITLトレーニングを実行できるように変更。
事前学習済みモデルとしてMRI_TOMのベストモデル（Val Dice: 0.9309）を使用。

### 変更元ファイル (Source files)
- `server/config.py`
- `server/main.py`
- `server/trainer.py`
- `server/inference.py`

### Before / After 比較

| 項目 | Before | After |
|------|--------|-------|
| PRETRAINED_PATH | `pytorch/pretrained.pt`（ローカル） | MRI_TOM `best_vanillaunet_model.pth` (Val Dice 0.9309) |
| 入力形式 | RGB 3ch + ImageNet正規化 | Grayscale 1ch + [0,1]正規化 |
| 損失関数 | DiceBCELoss (binary) | MultiClassDiceCELoss (CE + Dice, 背景除外) |
| マスク読み込み | 赤色バイナリマスク | インデックスカラーPNG → label_config.jsonカラーマップ |
| Dice評価 | binary dice | per-class dice (背景除外平均) |
| 推論出力 | 赤色RGBA (血管=赤, 背景=透明) | マルチクラスRGBA (label_config色, 背景=透明) |
| 推論前処理 | RGB 3ch + ImageNet正規化 + circle mask | Grayscale 1ch + [0,1]正規化 |
| 起動時モデル初期化 | なし | PRETRAINED→best.pth自動コピー（初回のみ） |
| 学習後CoreML変換 | 手動 | 自動変換（学習完了後） |

### 変更理由 (Reason for change)
- 眼窩MRI 10クラスセグメンテーション用にパイプラインを適合
- iPadとサーバー間のモデル共有フロー確立
- HITLループ（iPad annotation → サーバー再学習 → CoreML変換 → iPadダウンロード）の完成

### モデル選択根拠
| モデル | Val Dice | パラメータ数 |
|--------|----------|-------------|
| vanilla_unet_resnet34 (best) | **0.9309** | 17.3M |
| UNet resnet34 (標準) | 0.7934 | 24.4M |

→ vanilla_unet_resnet34の`best_vanillaunet_model.pth`を採用（Dice +17.4%、パラメータ数-29%）

### 技術メモ
- MRI_TOMチェックポイント形式: `model_state_dict` + `optimizer_state_dict` + `metrics` 等
- 当サーバーのチェックポイント形式も `model_state_dict` キーで統一 → 互換性OK
- トレーニングでは PRETRAINED_PATH → best.pth の順でフォールバック初期化
- label_config.json のカラーマップ: 各症例ディレクトリに配置、trainer/inferenceで参照

### ハマった点と解決策 (Troubleshooting)

#### 1. バイナリ→マルチクラス損失関数の移行
- **問題**: DiceBCELoss (sigmoid + binary CE) はマルチクラスに適用不可
- **解決**: `MultiClassDiceCELoss` を新規実装
  - `F.cross_entropy` (マルチクラスCE) + per-class Dice Loss（背景class=0を除外）
  - 背景除外が重要：背景は面積が大きくDiceが常に高いため、含めると学習が甘くなる

#### 2. マスク形式の変換
- **問題**: iPadからのマスクはインデックスカラーPNG（RGBAで各クラスに固有色）、PyTorchのCrossEntropyLossは`long`型の(B,H,W)を期待
- **解決**: `label_config.json`のカラーマップを参照して`(R,G,B) → class_id`変換を実装
  - `_color_mask_to_index()`: RGBAマスク → int64インデックス配列
  - case_idごとにカラーマップをキャッシュ（同一症例で毎回JSONを読まない）

#### 3. Grayscale入力の正規化
- **問題**: 旧パイプラインはRGB 3ch + ImageNet正規化だが、vanilla_unetは1ch grayscale
- **解決**: 単純な`pixel/255.0`正規化に変更（MRI_TOMの学習時と同じ）
  - iPad側 `createGrayscaleInputArray()` も同じ正規化を使用 → サーバーとiPadで推論結果が一致

#### 4. best.pthの初回配置
- **問題**: サーバー初回起動時にモデルが無いと推論APIが503を返す
- **解決**: 起動時に `PRETRAINED_PATH` が存在し `best.pth` が無ければ自動コピー
  - 2回目以降はHITL学習済みモデルが `best.pth` にあるのでスキップ

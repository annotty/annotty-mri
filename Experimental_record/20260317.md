# 2026-03-17 実験記録
## UNetService Inference処理修正（1ch/マルチクラス対応）

### 概要 (Overview)
サーバーのVanillaUNet（1ch grayscale / `/255.0`正規化 / 10クラス / 256x256）に対応するため、
iPad側 `UNetService` の前処理・後処理をCoreMLモデルのshapeから自動検出して切り替える仕組みに修正。

### 変更元ファイル (Source file)
- `Annotty-MRI/Services/UNet/UNetService.swift`
- `Annotty-MRI/ViewModels/CanvasViewModel.swift`

### Before / After 比較
| 項目 | Before | After |
|------|--------|-------|
| 入力サイズ | `static let inputSize = 512` 固定 | `modelConfig.inputSize` (CoreMLから自動検出) |
| 入力チャンネル | 常に3ch RGB | 1ch or 3ch 自動検出。1ch時はRGB→gray変換+/255.0 |
| 正規化 | downloaded時もImageNet mean/std適用 | 1ch: `/255.0`のみ、3ch: ImageNet norm |
| 後処理 | sigmoid→0.5閾値→バイナリ固定 | `numClasses==1`: sigmoid閾値、`>1`: argmax→クラスID |
| UNetMaskResult | `mask + size` | `mask + size + isMultiClass` |
| applyUNetMask | `(unetMask, size)` → currentClassIDに変換 | `(UNetMaskResult)` → multi-classならクラスIDそのまま |
| upscale | bilinear補間のみ | binary: bilinear、multi-class: nearest-neighbor |

### 変更理由 (Reason for change)
サーバーモデルは1chグレースケール入力・10クラス出力で学習済みだが、iPad側は旧SMP（3ch RGB/ImageNet正規化/バイナリ）前提のままだった。
`modelSource`フラグによる分岐もImageNet前提で、ダウンロードモデルに誤った前処理を適用していた。

### 追加メソッド
- `ModelConfig` struct: `inputChannels`, `inputSize`, `numClasses`
- `detectModelConfig(from:)`: CoreML `modelDescription` からshape読み取り
- `upscaleMaskNearestNeighbor`: マルチクラスmask用（クラスIDを破壊しない）

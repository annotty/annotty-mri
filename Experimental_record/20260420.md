### 変更理由 (Reason for change)
- **TOM500 学習時と前処理を統一**：HITL の再学習で pretrained の表現を活かすため、正規化方式を TOM 側と揃える必要があった。
- **ケース単位 Group K-fold**：同一症例のスライスが train/val に混在するとデータリークで過大評価。
- **WSL 経由 CoreML 変換**：`coremltools` は Windows 未対応なので、Windows サーバ上では WSL で変換を実行する必要がある。
- **`IMAGE_SIZE=512` / `MODEL_INPUT_SIZE=256` 分離**：API は 512 で I/O、モデルは 256 で推論。Swift 側の表示解像度を維持しつつモデル学習コストを抑える。

### ハマった点と解決策 (Troubleshooting)
- **改行コードが全ファイル CRLF 化**：Windows で編集されたため、そのまま cp すると git diff が全行変更で汚くなる。`tr -d '\r'` でコピー時に LF に変換し、実質変更のあるファイルのみ差分として記録した。
- **`static/` ディレクトリが空**：`main.py` は `os.listdir(STATIC_DIR)` で空チェックしているため、コピー不要と判断。
- **Experimental_record も Windows 側で増えていた**（20260410.md, 20260417.md）：今回は server 更新スコープのためコピー保留。必要に応じて別途取り込む。

### Mac 環境での実行上の注意
- `trainer.py` の `device = torch.device("cuda")` は **CUDA 固定**。Mac では学習不可（推論は inference.py 側で CPU/GPU 自動選択）。
- `main.py` は `/train` 完了後に `convert_to_coreml_wsl()` を呼ぶ。`wsl` コマンドが無い Mac では失敗する（`D:\Annotty_MRI\convert_coreml_wsl.sh` も存在しない）。
- `TOM_PNG_DIR=D:\TOM500\dataset\train_png` は Windows パス。Mac では `os.path.isdir` が False となり混合学習はスキップされる（既存動作に影響なし）。
- `PRETRAINED_PATH=data/models/pytorch/pretrained.pt` は相対パス化されたため、Windows から pretrained モデルを配置する必要がある（Mac 側の `~/MRI_TOM/...` からはロードされなくなった）。

# 2026-04-11 実験記録

## ペン太さ slider=1 を本当に1pxにする修正

### 概要 (Overview)
アノテーションのペン太さスライダーで `1` を選んでも実際には4〜6pxの太い線が描かれていた問題を修正。`brushRadius` の意味を「半径」から「線幅（直径）」に変更し、Metalシェーダ呼び出しで誤って掛けられていた `contentScaleFactor` を除去した。

### 変更元ファイル (Source files)
- `Annotty-MRI/Metal/MetalRenderer.swift`
- `Annotty-MRI/ViewModels/CanvasViewModel.swift`
- `Annotty-MRI/Views/CanvasView/SmoothStrokeOverlay.swift`

### Before / After 比較
| 項目 | Before | After |
|------|--------|-------|
| `brushRadius` のセマンティクス | 半径（image pixel） | 線幅 / 直径（image pixel） |
| `MetalRenderer.applyStamp` の半径計算 | `radius × maskScaleFactor × contentScaleFactor` | `radius × 0.5 × maskScaleFactor` |
| slider=1 のとき実際の線幅 (Retina, maskScaleFactor=2) | mask半径4 → 元画像で約4–6px | mask半径1 → 元画像で約1px |
| `brushPreviewSize` | `brushRadius × 2 × currentScale` | `brushRadius × currentScale` |
| Smooth stroke overlay の `lineWidth` | `brushRadius × 2` | `brushRadius` |
| bbox padding (CanvasViewModel 884/2030/2051) | `brushRadius × maskScaleFactor` | `brushRadius × 0.5 × maskScaleFactor` |

### 変更理由 (Reason for change)
1. `MetalRenderer` は `params.center` を mask 座標で渡しているのに、`adjustedRadius` 側で `contentScaleFactor`（Retina=2 / iPad Pro=3）を二重に掛けていた → ストロークが2〜3倍太くなる根本原因。
2. UI 上の "1" は半径ではなく「線の太さ 1px」をユーザは期待する。スライダの値をそのまま線幅とした方が直感的。

### ハマった点と解決策 (Troubleshooting)
- `brushRadius` を参照している箇所が複数ある（プレビュー、bbox padding、smooth overlay、ステップ間隔）。半径→線幅の変更で「半分」を入れる必要のある箇所と、入れてはいけない箇所を切り分けた:
  - 実際の Metal シェーダの半径: `× 0.5` 必要
  - bbox padding: `× 0.5` 必要（半径相当だから）
  - プレビュー直径: `× 2` を取り除く
  - smooth overlay の `lineWidth`: `× 2` を取り除く
  - 補間ステップ間隔 `brushRadius × 0.3`: 数値的にそのまま意味が通るので変更不要
- デフォルト値 `brushRadius = 20` は変更しなかった。意味としては「20px 半径」→「20px 線幅」に変わるため、デフォルトのペンが約半分の太さに感じられる点に注意。

## 追記: 左パネルのブラシ見本サイズが実際の2倍だった問題

### 概要 (Overview)
左パネルのブラシプレビュー円が実際の描画線幅の2倍（iPad Proなら3倍）に表示されていた。SwiftUIの `.frame()` は **point** 単位だが、`brushPreviewSize` が **screen pixel** で計算されていたのが原因。

### 変更元ファイル
- `Annotty-MRI/ViewModels/CanvasViewModel.swift`（`brushPreviewSize`）
- `Annotty-MRI/Views/CanvasView/CanvasContainerView.swift`（`SmoothStrokeOverlay` への引き渡し）

### Before / After
| 項目 | Before | After |
|------|--------|-------|
| `brushPreviewSize` | `brushRadius × currentScale` (= screen pixel) | `brushRadius × currentScale / contentScaleFactor` (= point) |
| `SmoothStrokeOverlay` の `brushRadius` 引数 | `viewModel.brushRadius × currentScale` | `viewModel.brushPreviewSize`（同一式を再利用） |

### 変更理由
`currentScale` は `fitToView(viewSize: drawableSize)` で初期化されており、drawableSize は **pixel** 単位。よって `image_pixel × currentScale = screen_pixel`。SwiftUI に渡す際は `contentScaleFactor` で割って point に変換しないと Retina で2倍、iPad Pro で3倍の見本サイズになる。

### ハマった点と解決策
- 「半分にする」だけで済む（×0.5 を入れる）デバイスもあれば、iPad Pro のように 1/3 が正しいデバイスもある。ハードコードの `* 0.5` ではなく `renderer?.contentScaleFactor` を参照する形にして汎用化した。
- canvas 上のドロー中プレビュー円（`CanvasContainerView` 30行目）と smooth ストロークオーバーレイの両方が同じ式を使うべきなので、smooth 側は `brushPreviewSize` を再利用するように整理した。

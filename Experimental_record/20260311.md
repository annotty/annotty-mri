# 2026-03-11 実験記録

## GitHub master との差分修正（データ出力形式の復元）

### 概要 (Overview)
ローカルで変更していたデータ出力形式を、GitHub master (`annotty/annotty-mri`) に合わせて復元した。
主な変更は `exportMaskForServer()` の出力形式とラベル設定の受け渡し方式。

### 変更元ファイル (Source file)
- `Annotty-MRI/Services/HIL/HILServerClient.swift`
- `Annotty-MRI/ViewModels/HILViewModel.swift`
- `Annotty-MRI/ViewModels/CanvasViewModel.swift`
- `Annotty-MRI/Views/MainView.swift`

### Before / After 比較

| 項目 | Before (ローカル変更) | After (GitHub master準拠) |
|------|----------------------|--------------------------|
| マスク出力サイズ | 256x256 | 512x512 |
| マスク出力形式 | グレースケール index PNG (pixel値=classID) | RGBA カラー PNG (クラス色で着色) |
| マスク色空間 | `CGColorSpaceCreateDeviceGray()` | `CGColorSpaceCreateDeviceRGB()` |
| バイト/ピクセル | 1 | 4 |
| ラベル設定struct名 | `LabelClassConfig` | `LabelClassInfo` |
| ラベル設定取得メソッド | `fetchLabelConfig()` | `downloadLabelConfig()` |
| ラベル設定適用方式 | `applyServerLabelConfig()` でメモリ直接適用 | `ProjectFileService` 経由でファイルI/O → `loadLabelConfigFromProject()` |
| ImageListResponse | `labelConfig` フィールドなし | `labelConfig: LabelConfigResponse?` あり |
| connect() | 引数なし、別途 `fetchLabelConfig` | `canvasVM:` 引数あり、レスポンス内の `labelConfig` を利用 |
| MainView Dashboard dismiss | `applyLabelConfig(to:)` 呼び出しあり | `onDismiss` なし |

### 変更理由 (Reason for change)
- サーバー側が RGBA カラー PNG (512x512) を期待しているため、互換性を維持する必要があった
- ラベル設定の永続化にはファイルI/O経由の方が再起動後も設定が残り堅牢

### ハマった点と解決策 (Troubleshooting)
- `HILViewModel.swift` の `connect()` メソッドシグネチャ変更に伴い、呼び出し元（`HILDashboardView`, `MainView`）も確認が必要だった
- GitHub masterでは `connect(canvasVM: CanvasViewModel? = nil)` とデフォルト引数があるため、既存の `connect()` 呼び出しはそのまま動作
- `applyServerLabelConfig` 削除時、`HILViewModel` 内の参照と `MainView` の `onDismiss` 内の呼び出しも合わせて削除が必要だった

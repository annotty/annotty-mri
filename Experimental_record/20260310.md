# 2026-03-10 実験記録

## サーバーからのクラス情報ダウンロード機能実装

### 概要 (Overview)
サーバーの `GET /label_config` エンドポイントからクラス定義（名前・色・ID）を取得し、iPad側に反映する機能を実装した。

### 変更元ファイル (Source file)
- `Annotty-MRI/Services/HIL/HILServerClient.swift`
- `Annotty-MRI/ViewModels/CanvasViewModel.swift`
- `Annotty-MRI/ViewModels/HILViewModel.swift`
- `Annotty-MRI/Views/MainView.swift`

### Before / After 比較
| 項目 | Before | After |
|------|--------|-------|
| HILServerClient | label_configエンドポイント未対応 | `fetchLabelConfig()` メソッド + `LabelConfigResponse` モデル追加 |
| CanvasViewModel | サーバーからのクラス適用メソッドなし | `applyServerLabelConfig()` メソッド追加 |
| HILViewModel | クラス情報取得なし | `connect()` 時に自動取得 + `applyLabelConfig(to:)` で適用 |
| MainView | Dashboard閉じ時にクラス適用なし | `onDismiss` でクラス情報を自動適用 |

### 変更理由 (Reason for change)
- SPEC.mdの「label_config.jsonによる双方向クラス定義」仕様に準拠
- サーバーで定義されたクラス（外直筋、内直筋、視神経等）がiPadに反映されるようにする

### ハマった点と解決策 (Troubleshooting)
- HILDashboardViewはcanvasVMを直接持っていないため、Dashboard内での適用は不可
- 解決策: `importImages` 内での適用 + MainView の `onDismiss` での適用の二段構え

## リモートリポジトリとの同期 - exportMaskForServer修正

### 概要 (Overview)
GitHubリモートリポジトリ (annotty/annotty-mri) とローカルコードの差分を確認し、`exportMaskForServer()` 関数をリモート版に合わせて修正した。

### 変更元ファイル (Source file)
- `Annotty-MRI/ViewModels/CanvasViewModel.swift` (line 1929-1980)

### Before / After 比較
| 項目 | Before | After |
|------|--------|-------|
| 出力解像度 | 512x512 | 256x256 |
| ピクセルフォーマット | RGBA (4 bytes/pixel) | Grayscale (1 byte/pixel) |
| クラスエンコーディング | バイナリ: classID > 0 → 赤(255,0,0,255) | インデックス: ピクセル値 = クラスID (0=背景, 1-9=クラス) |
| カラースペース | CGColorSpaceCreateDeviceRGB() | CGColorSpaceCreateDeviceGray() |
| アルファ情報 | premultipliedLast | none |
| データサイズ (概算) | ~1MB | ~65KB |

### 変更理由 (Reason for change)
- SPEC.mdの「インデックスカラーPNG」仕様に準拠するため
- マルチクラスセグメンテーション情報を保持するため（Before版ではクラス情報が消失していた）
- サーバー側で `np.array(Image.open(png))` → NIfTIラベルマップに直接挿入可能にするため

### ハマった点と解決策 (Troubleshooting)
- 特になし。リモートとローカルの差分は `exportMaskForServer()` 関数のみだった
- その他の全Swiftソースファイル（RightPanelView, ImageSettingsOverlayView, ProjectFileService, MaskClass, AnnotationProject, MainView, TopBarView）は同一であることを確認
